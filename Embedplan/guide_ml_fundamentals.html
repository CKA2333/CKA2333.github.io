1
ML 是什么 — 用硬件工程师的视角理解
20 分钟

你已经写过滤波器（LPF、IIR）、PID 控制器、卡尔曼滤波。这些都是手动设计的系统：你知道物理规律，把它编码成数学公式。

ML 是什么：当你不知道"规律"长什么样，让程序自动从数据里找出来。

🔧 嵌入式类比

ADC 采集振动数据 → 你手写了一个 FFT 阈值判断是否异常。
ML 版本：给它 10000 条"正常振动"和"故障振动"的样本，让它自动学到哪种频谱对应哪种状态。
你不用知道哪个频率是关键，模型会自己找到。

ML 能做什么（嵌入式常见场景）

  • 分类：这段传感器数据是"行走/跑步/跌倒"中的哪种（IMU + 手势识别）
  • 异常检测：电机振动是否超出正常范围（预测性维护）
  • 关键词识别：麦克风听到 "Hey Siri" → 唤醒（Always-on DSP 芯片）
  • 回归：根据电流/温度预测剩余电量（BMS）
💡 Embedded + ML 岗位要求的 ML 知识

你不需要设计新网络架构，但要能：训练一个小模型 → 量化 → 导出为 TFLite / ONNX → 在 MCU 上运行推理 → 测量延迟和精度。这个流程你能 demo 出来，你就赢了。

核心词汇表（10 个最重要的概念）
30 分钟
Feature（特征）
模型的输入变量。例如：温度、电压、FFT 频率分量。向量 x = [x₁, x₂, ..., xₙ]。
Label（标签）
你想预测的答案。分类时是类别（0/1/2），回归时是连续值（如温度 23.5°C）。
Model（模型）
一组参数（weights + bias）定义的函数 f(x) = y_pred。训练前随机初始化，训练后参数有意义。
Loss（损失）
衡量预测值 y_pred 和真实值 y 之间差距的标量。训练目标 = 让 Loss 越来越小。
Gradient（梯度）
Loss 对每个参数的偏导数。告诉你"把这个参数调大/调小，Loss 会怎么变化"。
Learning Rate（学习率）
每次更新参数的步长。太大→震荡不收敛；太小→收敛慢。典型值 1e-3 到 1e-4。
Epoch
把整个训练集过一遍叫一个 epoch。通常训练 10–1000 个 epoch。
Batch Size
每次更新参数用多少条数据。典型值 32 / 64 / 128。越小更新越频繁但噪声大。
Overfitting（过拟合）
在训练集上表现好但测试集上差。模型"背答案了"而不是"学到规律"。
Activation（激活函数）
加入非线性，让神经网络能学复杂函数。常见：ReLU（max(0,x)）、Sigmoid（1/(1+e⁻ˣ)）。
梯度下降更新公式：
w = w - lr × ∂Loss/∂w
b = b - lr × ∂Loss/∂b

直觉：梯度指向 Loss 增大的方向，减去梯度就是往 Loss 减小的方向走。
3
手写线性回归（NumPy，从零）
45 分钟 · 跟着敲一遍！

任务：用传感器温度预测电池剩余电量（简化版 BMS 问题）。已有 6 组测量数据，训练一个 y = w*x + b 的线性模型。

数据
x=温度, y=电量%
→
前向传播
y_pred = w*x + b
→
计算 Loss
MSE = mean((y_pred-y)²)
→
反向传播
计算梯度 dw, db
→
更新参数
w -= lr * dw

完整代码（复制到 Python 文件并运行）

"""
linear_regression.py
任务：根据温度预测电池剩余电量
直接运行：python linear_regression.py
"""
import numpy as np

# ── 1. 数据（6 组温度 → 电量% 的测量值）──────────────────────
X = np.array([10.0, 20.0, 25.0, 30.0, 35.0, 40.0])  # 摄氏度
y = np.array([95.0, 90.0, 85.0, 78.0, 70.0, 60.0])  # 电量%
# 真实规律：温度越高，电量衰减越快（非线性，但用线性近似）

# ── 2. 初始化参数（随机猜一个起点）────────────────────────────
w = 0.0   # weight（斜率）
b = 0.0   # bias（截距）
lr = 0.001  # learning rate
n = len(X)

# ── 3. 训练循环 ────────────────────────────────────────────────
for epoch in range(2000):
    # 前向传播：用当前 w, b 计算预测值
    y_pred = w * X + b                       # shape: (6,)

    # 计算 MSE Loss
    loss = np.mean((y_pred - y) ** 2)

    # 反向传播：手动推导梯度
    # dLoss/dw = (2/n) * sum((y_pred - y) * X)
    # dLoss/db = (2/n) * sum(y_pred - y)
    dw = (2 / n) * np.dot((y_pred - y), X)
    db = (2 / n) * np.sum(y_pred - y)

    # 梯度下降：沿梯度反方向更新参数
    w -= lr * dw
    b -= lr * db

    if epoch % 500 == 0:
        print(f"Epoch {epoch:4d}: Loss={loss:.4f}  w={w:.4f}  b={b:.4f}")

# ── 4. 结果 ────────────────────────────────────────────────────
print(f"\n✓ 学到的模型: 电量 = {w:.3f} × 温度 + {b:.3f}")
print(f"  预测 28°C 时电量: {w*28+b:.1f}%")  # 应在 78-85% 之间

# ── 5. 预期输出（大约）────────────────────────────────────────
# Epoch    0: Loss=6272.5417  w=4.7   b=0.23
# Epoch  500: Loss=12.345     w=-1.12 b=107.5
# Epoch 2000: Loss=4.892      w=-1.17 b=109.6
# ✓ 学到的模型: 电量 = -1.17 × 温度 + 109.6
# 预测 28°C 时电量: 77.1%   ← 合理！
🔍 关键理解：梯度推导

Loss = mean((wx+b - y)²)。对 w 求偏导：链式法则 → ∂Loss/∂w = (2/n)·Σ(y_pred - y)·x。这就是代码里 dw = (2/n) * np.dot((y_pred-y), X) 的来源。手写一遍才能真正理解，以后看 PyTorch 的 loss.backward() 就知道它在干什么了。

⚠️ 常见错误

lr 太大（如 0.1）：Loss 会爆炸到 inf，因为每次迈的步子太大，反而越走越偏。如果看到 NaN，第一反应就是降 lr。

手写神经网络前向传播（NumPy）
45 分钟 · 跟着敲

神经网络 = 多层线性变换 + 激活函数。前向传播就是把输入从左往右传，每层做一次矩阵乘法然后加激活函数。

任务：2 个传感器读数（温度 + 电流） → 判断电池是否"健康"（二分类）

Layer 1: z1 = X @ W1 + b1 (线性变换)
a1 = ReLU(z1) (激活函数，加入非线性)
Layer 2: z2 = a1 @ W2 + b2 (线性变换)
out = Sigmoid(z2) (压缩到 0-1，当作概率) 
"""
neural_net_forward.py
2层神经网络前向传播：2个特征 → 3个隐藏神经元 → 1个输出（概率）
"""
import numpy as np

# ── 激活函数 ──────────────────────────────────────────────────
def relu(x):
    return np.maximum(0, x)    # 负数变0，正数不变

def sigmoid(x):
    return 1 / (1 + np.exp(-x))  # 把任意实数压缩到 (0, 1)

# ── 初始化参数（真实训练时用 Xavier/He 初始化）─────────────────
np.random.seed(42)
W1 = np.random.randn(2, 3) * 0.1  # 2个输入 → 3个隐藏神经元
b1 = np.zeros(3)
W2 = np.random.randn(3, 1) * 0.1  # 3个隐藏 → 1个输出
b2 = np.zeros(1)

# ── 前向传播函数 ──────────────────────────────────────────────
def forward(x):
    # Layer 1
    z1 = x @ W1 + b1          # (2,) @ (2,3) + (3,) = (3,)
    a1 = relu(z1)             # 非线性激活
    # Layer 2
    z2 = a1 @ W2 + b2          # (3,) @ (3,1) + (1,) = (1,)
    out = sigmoid(z2)          # 输出概率
    return out, (z1, a1, z2)   # 返回中间值用于调试

# ── 测试 ──────────────────────────────────────────────────────
# 输入：温度=35°C（归一化后0.7），电流=2.1A（归一化后0.6）
x_test = np.array([0.7, 0.6])
prob, intermediates = forward(x_test)
print(f"输出概率: {prob[0]:.4f}")
print(f"预测: {'健康' if prob[0] > 0.5 else '异常'}")
print(f"隐藏层激活: {intermediates[1]}")

# ── 批量输入（实际训练时用矩阵，一次处理多条数据）──────────────
X_batch = np.array([
    [0.7, 0.6],   # 样本1
    [0.2, 0.3],   # 样本2
    [0.9, 0.8],   # 样本3
])  # shape: (3, 2)

# 矩阵运算：一次前向传播处理 3 条数据
Z1_batch = X_batch @ W1 + b1   # (3,2)@(2,3) = (3,3)
A1_batch = relu(Z1_batch)
Z2_batch = A1_batch @ W2 + b2  # (3,3)@(3,1) = (3,1)
out_batch = sigmoid(Z2_batch)
print(f"\n批量输出:\n{out_batch}")
💡 为什么需要激活函数？

没有激活函数，多层线性变换还是线性变换（W2 @ W1 @ x 等价于 W_new @ x）。ReLU 引入非线性，让网络能学习弯曲的决策边界，否则无论多深都只是线性分类器。

任务一：PyTorch 版线性回归（和 Step3 完全同一个问题）

"""
pytorch_linear.py — 和 Step3 完全相同的任务，但用 PyTorch 自动求梯度
"""
import torch
import torch.nn as nn

# ── 数据 ──────────────────────────────────────────────────────
X = torch.tensor([[10.], [20.], [25.], [30.], [35.], [40.]])
y = torch.tensor([[95.], [90.], [85.], [78.], [70.], [60.]])

# ── 模型（nn.Linear 就是 y = w*x + b）────────────────────────
model = nn.Linear(1, 1)              # 1个输入，1个输出
loss_fn = nn.MSELoss()               # Mean Squared Error
optimizer = torch.optim.SGD(model.parameters(), lr=0.001)

# ── 训练循环 ──────────────────────────────────────────────────
for epoch in range(2000):
    y_pred = model(X)               # 前向传播
    loss = loss_fn(y_pred, y)        # 计算 Loss

    optimizer.zero_grad()            # 清空上次的梯度！（必须）
    loss.backward()                  # 自动反向传播，计算所有梯度
    optimizer.step()                 # 更新参数 w, b

    if epoch % 500 == 0:
        print(f"Epoch {epoch}: Loss={loss.item():.4f}")

w = model.weight.item()
b = model.bias.item()
print(f"\n✓ 学到的模型: 电量 = {w:.3f} × 温度 + {b:.3f}")
print(f"  预测 28°C: {w*28+b:.1f}%")

任务二：PyTorch 神经网络（2 层，手势分类）

"""
pytorch_nn.py — 用 nn.Sequential 快速搭建 2 层 NN
任务：4 个 IMU 特征 → 3 类手势（静止/挥手/握拳）
"""
import torch
import torch.nn as nn

# ── 模拟 IMU 数据（实际用你的传感器数据替换）──────────────────
torch.manual_seed(42)
X_train = torch.randn(120, 4)      # 120 条样本，4 个特征
y_train = torch.randint(0, 3, (120,)) # 标签 0/1/2

# ── 搭建神经网络 ──────────────────────────────────────────────
model = nn.Sequential(
    nn.Linear(4, 16),   # 4个特征 → 16个隐藏神经元
    nn.ReLU(),
    nn.Linear(16, 8),   # 16 → 8
    nn.ReLU(),
    nn.Linear(8, 3),    # 8 → 3类（不加 Softmax，CrossEntropyLoss 自带）
)

loss_fn = nn.CrossEntropyLoss()  # 多分类标准 Loss
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# ── 训练 ──────────────────────────────────────────────────────
for epoch in range(200):
    logits = model(X_train)
    loss = loss_fn(logits, y_train)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

# ── 推理（模拟在 MCU 上的推理过程）────────────────────────────
model.eval()                        # 关掉 dropout/batchnorm 等
with torch.no_grad():              # 推理时不需要梯度，省内存
    x_new = torch.tensor([[0.5, -0.2, 1.1, 0.3]])
    logits = model(x_new)
    pred = logits.argmax().item()
    print(f"预测类别: {pred} ({'静止' if pred==0 else '挥手' if pred==1 else '握拳'})")

# 模型大小（嵌入式很重要！）
total_params = sum(p.numel() for p in model.parameters())
print(f"模型参数量: {total_params}  ≈ {total_params*4/1024:.1f} KB (fp32)")
💡 三行 PyTorch 训练必须按顺序

zero_grad() → backward() → step()。如果忘了 zero_grad，梯度会累积，相当于 batch size 翻倍，Loss 会奇怪地震荡。这是最常见的 PyTorch bug 之一。

7
从模型到嵌入式：量化、TFLite、ONNX
40 分钟 · TinyML 核心！

在 MCU 上跑 ML 的核心挑战：内存有限（几十 KB RAM）、没有 FPU 或 FPU 性能弱、没有操作系统。

量化（Quantization）— 最重要的概念

训练时用 float32（4 字节/参数），部署到 MCU 用 int8（1 字节/参数）。模型缩小 4 倍，推理速度提升 2–4 倍，精度损失通常 < 1%。

float32 权重 0.1234 → int8 量化
量化公式：x_int8 = round(x_float32 / scale) + zero_point
反量化：x_float32 ≈ (x_int8 - zero_point) × scale

导出为 TFLite（用于 TensorFlow Lite Micro）

"""
export_tflite.py — PyTorch 模型 → ONNX → TFLite (通过转换工具链)
或者直接用 TensorFlow 训练再导出 TFLite
这里演示 TensorFlow 路线（更直接）
"""
import tensorflow as tf
import numpy as np

# 假设你有一个训练好的 Keras 模型
model = tf.keras.Sequential([
    tf.keras.layers.Dense(16, activation='relu', input_shape=(4,)),
    tf.keras.layers.Dense(8,  activation='relu'),
    tf.keras.layers.Dense(3,  activation='softmax'),
])
model.compile(optimizer='adam', loss='sparse_categorical_crossentropy')
# model.fit(X_train, y_train, epochs=50)  # 训练

# ── 导出为 TFLite（INT8 量化）────────────────────────────────
converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]  # 自动量化

# 提供 representative dataset 以提高量化精度
def representative_dataset():
    for _ in range(100):
        yield [np.random.randn(1, 4).astype(np.float32)]

converter.representative_dataset = representative_dataset
tflite_model = converter.convert()

# 保存模型文件
with open('gesture_model.tflite', 'wb') as f:
    f.write(tflite_model)

print(f"模型大小: {len(tflite_model)/1024:.1f} KB")
# 这个简单模型量化后通常 < 5 KB，完全可以放入 MCU Flash

在 C++ 中用 TFLite Micro 运行推理（关键代码结构）

// 这是嵌入式端的 C++ 推理代码结构（伪代码）
// 完整教程见 guide_tinyml.html

// 1. 把 .tflite 转为 C 数组（xxd 命令或 Edge Impulse 自动生成）
// xxd -i gesture_model.tflite > model_data.cc

#include "tensorflow/lite/micro/micro_interpreter.h"
#include "model_data.h"

// 2. 分配 arena（MCU 上的推理内存池）
constexpr int kArenaSize = 8 * 1024;  // 8 KB
uint8_t arena[kArenaSize];

// 3. 创建解释器
tflite::MicroInterpreter interpreter(model, resolver, arena, kArenaSize);
interpreter.AllocateTensors();

// 4. 填入输入（传感器数据）
float* input = interpreter.input(0)->data.f;
input[0] = accel_x; input[1] = accel_y;
input[2] = accel_z; input[3] = gyro_z;

// 5. 运行推理
interpreter.Invoke();

// 6. 读取输出（3类的概率）
float* output = interpreter.output(0)->data.f;
int pred = std::max_element(output, output+3) - output;
