核心恒等式（XOR 最多）
XOR 恒等式
  • A ^ A = 0自己消掉自己 → 找唯一数的核心
  • A ^ 0 = A和 0 XOR 不变
  • A ^ B ^ A = B两个A可以消掉
  • A ^ B = B ^ A交换律
  • (A^B)^C = A^(B^C)结合律
AND / OR / NOT 恒等式
  • A & 0 = 0AND 0 全清零
  • A & 1 = AAND 1 保留（用于 mask）
  • A | 0 = AOR 0 不变
  • A | A = A幂等律
  • ~(~A) = A双 NOT 还原
位操作常用技巧（直接用在题目里）
取第 k 位的值
#191 Hamming Weight #137 Single Number II
// 取 num 的第 k 位（0-indexed，从右数） int bit = (num >> k) & 1; // 遍历所有 32 位 for (int k = 0; k < 32; k++) { int bit = (num >> k) & 1; }
原理：>>k 把第 k 位移到最右，&1 只保留最后一位
判断 n 是否是 2 的幂
#231 Power of Two #342 Power of Four
// 2的幂的二进制只有一个1：n & (n-1) 清掉最低位的1 // 8 = 1000, 7 = 0111, 8 & 7 = 0000 ✓ bool isPow2 = (n > 0) && (n & (n - 1)) == 0; // n & (-n) 孤立出最低位的1 // 如果 n == n & (-n)，说明只有一个1 bool isPow2v2 = (n > 0) && (n == (n & (-n)));
记忆法：2 的幂只有一个 1，n-1 会把那个 1 以及右边全变成 1，AND 之后全消掉
XOR 找唯一数
#136 Single Number
// 出现两次的数互相消掉（A^A=0），剩下就是答案 int solo = 0; for (int x : nums) solo ^= x; return solo; // [4,1,2,1,2] → 0^4^1^2^1^2 = 4
按位计数 mod 3（出现三次的数）
#137 Single Number II
int bits[32] = {0}; for (int num : nums) // 每个数字 for (int k = 0; k < 32; k++) // 每一位 bits[k] += (num >> k) & 1; int res = 0; for (int k = 32; k >= 0; k--) // 重新组合 res |= (bits[k] % 3) << k; // 用 |= 不用 += return res;
为什么 |= 不用 +=：用加法在 bit 31 处会 overflow 进位破坏其他 bit；|= 直接设置那个 bit，不会影响其他位。
32 哪来的：int 是 32-bit，-2³¹ ~ 2³¹-1 的范围就是 32 位的结果。

双指针模板
左右指针（对撞型）
#167 Two Sum II #11 Container With Water #15 3Sum
int i = 0, j = nums.size() - 1; // 两端出发 while (i < j) { int sum = nums[i] + nums[j]; if (sum == target) return {i, j}; else if (sum < target) i++; // 太小，左指针右移 else j--; // 太大，右指针左移 }
前提：数组必须已排序。往中间夹逼，每次排除一个不可能的值。
快慢指针（同向型）
#26 Remove Duplicates #27 Remove Element #283 Move Zeroes
// j = 慢指针（下一个写入位置） // i = 快指针（扫描用） int j = 0; for (int i = 0; i < nums.size(); i++) { if (nums[i] != 0) { // 找到目标 swap(nums[i], nums[j]); // 放到 j 位置 j++; // j 前进 } }
关键：j 始终 ≤ i，所以 swap 时 j 位置一定是"已处理完的废料"，不会破坏未读的数据。
滑动窗口
#3 Longest Substring #209 Min Subarray Sum #438 Find Anagrams
int left = 0, right = 0; while (right < nums.size()) { // 1. 扩展右边界（加入新元素） window.add(nums[right]); right++; // 2. 收缩左边界（窗口不合法时） while (window 不合法) { window.remove(nums[left]); left++; } // 3. 更新答案 ans = max(ans, right - left); }
窗口可以是 int（求和）或 unordered_map（统计字符频率）。右进左出，维护一个动态的合法区间。
从后往前合并（不需要额外空间）
#88 Merge Sorted Array
// nums1 末尾有空位，从后往前填，避免覆盖未读数据 int i = m - 1; // nums1 最后一个真实元素 int j = n - 1; // nums2 最后一个元素 int k = m + n - 1; // 写入位置 while (i >= 0 && j >= 0) { if (nums1[i] > nums2[j]) nums1[k--] = nums1[i--]; else nums1[k--] = nums2[j--]; } while (j >= 0) nums1[k--] = nums2[j--]; // nums2 剩余
为什么省空间：写入位置 k 永远在读指针 i 的右边，所以永远写的是已经读过或者空着的位置。
unordered_map vs map
unordered_map（哈希表，常用）
// 创建：key=数字，value=次数 unordered_map<int, int> m; // 增 / 改 m[key]++; // 计数（不存在时自动创建为0） m[key] = value; // 直接赋值 // 查 m[key] // 直接取值（会创建！） m.count(key) // 1 存在 / 0 不存在（推荐） m.find(key) != m.end() // 同上，更明确 // 删 m.erase(key);
查找/插入 O(1) 平均，但无序
map（红黑树，有序）
// 按 key 升序排列 map<int, int> m; // 操作和 unordered_map 完全一样 m[key]++; m.count(key); // 区别：有序，查找 O(log n) // Java 对应：TreeMap vs HashMap
查找/插入 O(log n)，但 key 自动排序
遍历 Map
// 方式1：auto（最常用） for (auto x : m) { x.first; // key x.second; // value } // 方式2：结构化绑定（C++17，更清晰） for (auto [key, val] : m) { // 直接用 key 和 val } // 方式3：显式类型 for (pair<int,int> x : m) { ... }
Map 常用题型模板
计数 + 找唯一 / 频率

unordered_map<int,int> m; for (int x : nums) m[x]++; // 统计次数 for (auto [k, v] : m) if (v == 1) return k; // 找出现一次的

Two Sum 经典用法（存下标）

单调队列（滑动窗口最大值）的核心工具
单调栈（最常考！）
下一个更大元素（单调递减栈）
#496 Next Greater Element I #739 Daily Temperatures #84 Largest Rectangle
// 739: 每天等几天才能看到更高温度 vector<int> ans(n, 0); stack<int> st; // 存下标（不存值！） for (int i = 0; i < n; i++) { // 当前比栈顶大 → 栈顶找到了"下一个更大" while (!st.empty() && T[i] > T[st.top()]) { ans[st.top()] = i - st.top(); // 等待天数 st.pop(); } st.push(i); // 当前下标入栈，等待被解决 } // 栈中剩余的没有更大值，初始化为0已覆盖
核心思想：栈里维护"等待被解决"的下标序列，保持栈底→栈顶单调递减。新元素 > 栈顶时弹出并记录答案。
记忆法：求"下一个更大" → 维护单调递减栈（从栈底到栈顶值递减）
BFS 模板（队列）
BFS 层序遍历 / 最短路（标准模板）
#102 Binary Tree Level Order #200 Number of Islands #127 Word Ladder
queue<int> q; // 或 queue<TreeNode*>、queue<pair<int,int>> q.push(start); visited[start] = true; int step = 0; while (!q.empty()) { int sz = q.size(); // 关键：固定当前层的节点数 for (int i = 0; i < sz; i++) { auto node = q.front(); q.pop(); if (node == target) return step; for (auto nb : neighbors(node)) { if (!visited[nb]) { visited[nb] = true; q.push(nb); } } } step++; }
关键：用 sz = q.size() 在 while 循环开始时固定本层大小，然后处理完 sz 个节点后 step++，这样 step 才能准确计层数（最短距离）。
单调队列（Deque）
滑动窗口最大值
#239 Sliding Window Maximum
// deque 存下标，保持 front 始终是窗口内最大值的下标 deque<int> dq; vector<int> res; for (int i = 0; i < n; i++) { // 1. 超出窗口的元素从左端踢出 if (!dq.empty() && dq.front() < i - k + 1) dq.pop_front(); // 2. 比当前小的从右端踢出（它们不可能再是最大值） while (!dq.empty() && nums[dq.back()] < nums[i]) dq.pop_back(); dq.push_back(i); // 3. 窗口满了才记录答案 if (i >= k - 1) res.push_back(nums[dq.front()]); }
单调队列 vs 单调栈：队列需要两端操作（新元素从右端进，过期元素从左端出），所以必须用 deque 而不是 stack。front 永远是当前窗口最大值。
结构定义 + 基础遍历
ListNode 结构（LeetCode 标准）
struct ListNode { int val; ListNode* next; ListNode(int x) : val(x), next(nullptr) {} }; // 遍历 ListNode* cur = head; while (cur != nullptr) { // 操作 cur->val cur = cur->next; }
Dummy Head（必背技巧）
// 在 head 前加一个哑节点，统一边界情况 ListNode dummy(0); dummy.next = head; ListNode* prev = &dummy; // prev 从 dummy 开始 // 操作完后返回 dummy.next（真正的头） return dummy.next;
何时用 Dummy：删节点 / 插节点 / 合并链表。避免 head 节点是特殊情况导致多写 if。
快慢指针（Floyd 算法）
找中点 + 检测环
#876 Middle of Linked List #141 Linked List Cycle #142 Cycle II（找入口）
// 找中点：slow 走一步，fast 走两步 ListNode* slow = head, *fast = head; while (fast != nullptr && fast->next != nullptr) { slow = slow->next; fast = fast->next->next; } // slow 是中点（偶数时是后半段起点） // 检测环：相遇 = 有环 while (fast && fast->next) { slow = slow->next; fast = fast->next->next; if (slow == fast) return true; // 有环 } return false; // 找环入口（#142）：相遇后把一个指针移到 head // 然后两个同速走，再次相遇处就是环入口
为什么 fast 一定能追上 slow：如果有环，fast 每次比 slow 多走一步，环内距离每轮减 1，最终差值变为 0（相遇）。
反转链表（迭代）
#206 Reverse Linked List #92 Reverse II（区间反转）
// 三指针：prev → cur → nxt ListNode* prev = nullptr; ListNode* cur = head; while (cur) { ListNode* nxt = cur->next; // 先保存下一个 cur->next = prev; // 反转指针方向 prev = cur; // prev 前移 cur = nxt; // cur 前移 } return prev; // prev 是新 head // 示意：null ← 1 ← 2 ← 3，prev 停在 3
画图最好记：初始 null ← | 1 → 2 → 3。每次把当前节点的 next 指向 prev（翻转一根箭头），然后三个指针整体右移一步。
合并两个有序链表
#21 Merge Two Sorted Lists #23 Merge K Sorted Lists
ListNode dummy(0); ListNode* tail = &dummy; while (l1 && l2) { if (l1->val <= l2->val) { tail->next = l1; l1 = l1->next; } else { tail->next = l2; l2 = l2->next; } tail = tail->next; } tail->next = l1 ? l1 : l2; // 把剩余的链表直接接上 return dummy.next;
Dummy + tail 指针是链表构建题的万能模板。tail 始终指向已构建链表的最末尾节点，每次把较小节点接到 tail 后面。
删除倒数第 N 个节点
#19 Remove Nth From End
// 快指针先走 n+1 步，然后两个同速直到 fast == null ListNode dummy(0); dummy.next = head; ListNode* fast = &dummy, *slow = &dummy; for (int i = 0; i <= n; i++) fast = fast->next; // n+1 步 while (fast) { fast = fast->next; slow = slow->next; } // slow 停在目标节点的前一个 slow->next = slow->next->next; // 跳过目标节点 return dummy.next;
为什么是 n+1 步而不是 n 步：我们需要 slow 停在目标节点的"前驱"，所以 fast 和 slow 之间的距离是 n+1，这样 fast 到 null 时 slow 刚好差一步到目标。
二分查找三种模板
标准二分（找确切值）
#704 Binary Search #35 Search Insert Position
// 左闭右闭 [l, r]，找确切值 int l = 0, r = nums.size() - 1; while (l <= r) { // ≤ 不是 < int mid = l + (r - l) / 2; // 防溢出，不用 (l+r)/2 if (nums[mid] == target) return mid; else if (nums[mid] < target) l = mid + 1; else r = mid - 1; } return -1; // 没找到时 l 就是插入位置（#35）
mid = l + (r-l)/2：若用 (l+r)/2，当 l+r 超过 INT_MAX（约 21 亿）时整数溢出。嵌入式里数组可能很大，要养成习惯。
找左边界 / 右边界
#34 Find First and Last Position #278 First Bad Version
// 找左边界（第一个 >= target 的位置） int l = 0, r = n; // r = n 开区间 while (l < r) { // < 不是 <= int mid = l + (r - l) / 2; if (nums[mid] < target) l = mid + 1; else r = mid; // 相等也往左收 } // l == r 是左边界（l == n 表示不存在） // 找右边界（最后一个 <= target 的位置） int l2 = 0, r2 = n; while (l2 < r2) { int mid = l2 + (r2 - l2) / 2; if (nums[mid] <= target) l2 = mid + 1; // 相等也往右收 else r2 = mid; } // r2 - 1 是右边界
记忆法：找左边界：nums[mid] == target 时让 r = mid（继续往左）；找右边界：让 l = mid+1（继续往右）。两种只差一行。
二分答案（在值域上二分，最难最重要）
#875 Koko Eating Bananas #1011 Ship Within D Days #410 Split Array Largest Sum
// 875: 以速度 k 吃香蕉，能在 h 小时内吃完的最小 k // 关键认知：k 越大越容易完成 → 单调性 → 可以二分 int l = 1, r = *max_element(piles.begin(), piles.end()); while (l < r) { int mid = l + (r - l) / 2; if (canFinish(piles, mid, h)) r = mid; // mid 可行，试更小 else l = mid + 1; // 太慢，增大 k } return l; bool canFinish(vector<int>& p, int k, int h) { int hrs = 0; for (int x : p) hrs += (x + k - 1) / k; // 向上取整 return hrs <= h; }
三步套路：① 确定值域 [l, r]；② 写 check(mid) 判断 mid 是否满足条件；③ 根据"越大越容易/越小越容易"决定向哪边收缩。
口诀：如果 mid 满足且要找最小值 → r = mid；如果要找最大值 → l = mid（此时 mid 要取 (l+r+1)/2 防死循环）。
unordered_set 快查
unordered_set 操作
unordered_set<int> s; // 插入 / 删除 s.insert(val); s.erase(val); // 查找（O(1) 平均） s.count(val) // 1 或 0（推荐） s.find(val) != s.end() // 同上 // 批量构建（从 vector） unordered_set<int> s(v.begin(), v.end());
典型使用场景
// 1. 去重 unordered_set<int> seen(v.begin(), v.end()); // 2. O(1) 判断存在性（代替排序+二分） if (seen.count(target - x)) { ... } // 3. 滑动窗口字符集 unordered_set<char> window; window.insert(s[right]); window.erase(s[left]);
最长连续序列（Hash Set O(n) 解法）
#128 Longest Consecutive Sequence
// 把所有数放 set，只从"序列起点"开始向右数 unordered_set<int> s(nums.begin(), nums.end()); int best = 0; for (int x : s) { if (!s.count(x - 1)) { // x-1 不在 set → x 是起点 int len = 1; while (s.count(x + len)) len++; best = max(best, len); } } return best;
为什么是 O(n)：每个数只会被"作为某个序列的成员"数一次。跳过非起点保证了不重复遍历，总操作次数 = 总元素数 = O(n)。
