1
硬件准备 + 购买建议
5 分钟阅读

你只需要买一块 ESP32 开发板，其余全是免费软件。

  • ESP32 DevKitV1（推荐） — $3–5 (AliExpress) / $8–12 (Amazon)。搜索 "ESP32 38-pin DevKit" 或 "ESP32-WROOM-32"
  • USB Micro-B 数据线 — 注意要数据线（不是充电线），很多充电线只有电源没有数据针。如果有 Android 旧手机线就够了
  • 电脑 — Windows/Mac/Linux 都可以
💡 购买建议

AliExpress 搜 "ESP32 DevKit" 最便宜，买 2 块以防万一（$6–8 两块）。如果急需，Amazon 有 Prime 隔天到的版本约 $10–12。

⚠️ 注意芯片版本

市面上有两种驱动芯片：CP2102（Silicon Labs）和 CH340（国产），两者驱动不同，Step 4 会分别讲安装方法。收到板子后看 USB 接口旁边的芯片型号。

安装 VS Code
5 分钟
1
访问 code.visualstudio.com，点击大蓝按钮下载
2
Windows：运行 .exe 安装包，全部默认，勾选 "Add to PATH" 和 "Open with Code"（右键菜单）
3
Mac：拖到 Applications 文件夹，首次打开右键→Open 绕过 Gatekeeper
4
打开 VS Code，界面左侧有 5 个图标（文件、搜索、版本控制、调试、扩展），后面主要用 扩展（Extensions） 图标
💡 已装 VS Code？

直接跳到 Step 3。不需要重装。

3
安装 PlatformIO IDE 扩展
10 分钟（含下载）
1
按 Ctrl+Shift+X（Windows）或 Cmd+Shift+X（Mac）打开扩展面板
2
搜索框输入 "PlatformIO IDE"，找到蓝色蚂蚁图标的官方扩展（作者 PlatformIO）
3
点击 Install。PlatformIO 会自动安装 Python 环境，需要 5–10 分钟，进度条在右下角
4
安装完成后，VS Code 右下角弹出 "Reload Required"，点击 Reload Now
5
重载后，左侧栏底部出现🐜（蚂蚁）图标，表示安装成功。点击它进入 PlatformIO Home
⚠️ 安装卡住了？

PlatformIO 需要下载 Python 和工具链，网络慢时可能超过 10 分钟。耐心等待，不要强制关闭。如果真的失败，删除扩展重新安装。

4
安装 ESP32 USB 驱动
5–10 分钟

ESP32 开发板通过 USB 转串口芯片连接电脑，需要对应驱动。

先辨认你的芯片型号：看板子 USB 接口旁边，小黑芯片上的字。常见两种：

CP2102（Silicon Labs）：

1
访问 Silicon Labs 驱动下载页
2
下载 "CP210x Universal Windows Driver"（Windows）或 Mac 版本，双击安装

CH340（国产芯片，AliExpress 板子常见）：

1
访问 WCH 官网 CH340 驱动
2
下载 CH341SER.EXE，双击安装，点击 INSTALL 按钮

验证驱动：

1
插入 ESP32，Windows：打开设备管理器（Win+X → Device Manager），在 "Ports (COM & LPT)" 下看到 "CP210x USB to UART" 或 "CH340" 后面有 COMx 编号
2
Mac：打开终端，输入 ls /dev/cu.*，看到 /dev/cu.SLAB_USBtoUART 或 /dev/cu.usbserial-xxx
💡 Mac Ventura 及以上

新版 Mac 系统可能需要允许驱动：系统设置 → 隐私与安全 → 允许驱动加载。

5
创建第一个 ESP32 项目
5 分钟（+ 首次构建下载 5–10 分钟）
1
点击左侧🐜图标 → 进入 PlatformIO Home → 点击 "New Project"
2
Name：输入 hello_esp32
3
Board：搜索框输入 "ESP32 Dev"，选择 "Espressif ESP32 Dev Module"
4
Framework：选 Arduino（入门最友好，对 C++ 兼容好）
5
Location：勾选 "Use default location" 或自定义路径
6
点击 Finish。首次创建会下载 ESP32 工具链（~200MB），需要 5–10 分钟，右下角有进度条
⚠️ 首次下载很慢？

这是正常的，PlatformIO 需要下载编译器（xtensa-gcc）。只有第一次慢，之后创建 ESP32 项目瞬间完成。

6
理解项目结构
3 分钟

PlatformIO 项目和 Arduino IDE 的 .ino 文件不同，结构更专业：

hello_esp32/ ├── src/ │ └── main.cpp ← 你的主程序，从这里开始写 ├── include/ ← 头文件 (.h)，自定义函数声明 ├── lib/ ← 本地库，放不在 registry 里的库 ├── test/ ← 单元测试文件 └── platformio.ini ← 项目配置：板子型号、库依赖、串口速率

打开 platformio.ini 看看内容：

[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
monitor_speed = 115200    ; 串口波特率，和代码里 Serial.begin() 保持一致

以后要添加库、改板子型号、设置串口速率，都在这个文件里改。

💡 和 Arduino IDE 的关系

src/main.cpp 里的写法和 Arduino .ino 几乎一样，都有 setup() 和 loop()，只是多了 #include <Arduino.h>。

7
写第一个程序：LED 闪烁 + 串口输出
5 分钟

打开 src/main.cpp，把内容替换为：

#include <Arduino.h>

// ESP32 内置 LED 通常在 GPIO 2（板子上标 "2" 或 "LED"）
const int LED_PIN = 2;

void setup() {
  Serial.begin(115200);          // 初始化串口，波特率 115200
  pinMode(LED_PIN, OUTPUT);      // 设 GPIO 2 为输出
  Serial.println("ESP32 started!");
}

void loop() {
  digitalWrite(LED_PIN, HIGH);   // LED 亮
  Serial.println("LED ON");
  delay(500);

  digitalWrite(LED_PIN, LOW);    // LED 灭
  Serial.println("LED OFF");
  delay(500);
}
⚠️ 不同板子 LED 引脚不同

如果 GPIO 2 没有内置 LED，尝试 GPIO 22 或 GPIO 13。也可以接外部 LED（LED + 220Ω 电阻 → GPIO 引脚）。

8
编译和烧录
3–5 分钟

VS Code 底部工具栏（PlatformIO toolbar）有几个关键按钮：

  • ✓ Build（Ctrl+Alt+B）：编译，检查代码有没有语法错误
  • → Upload（Ctrl+Alt+U）：编译 + 烧录到板子
  • 🔌 Serial Monitor（Ctrl+Alt+S）：打开串口查看输出
1
先点 ✓ Build，等 Terminal 里出现 SUCCESS
2
插上 ESP32，点 → Upload。看到 Connecting... 时，如果卡住超过 5 秒，按住板子上的 BOOT 按钮 直到开始传输
3
出现 Leaving... Hard resetting via RTS pin... 表示烧录成功
❌ "No serial port found" 错误

说明 PlatformIO 找不到 ESP32。检查：① USB 线是数据线 ② 驱动已安装（Step 4）③ platformio.ini 加 upload_port = COMx（Windows 填你的 COM 口号）

💡 Serial.print vs println

Serial.print() 不换行，Serial.println() 换行。调试时用 Serial.printf("value = %d\n", x) 格式化输出更方便。

10
常见问题排查
备查

问题 1：上传时一直显示 Connecting.........___

  • 按住 BOOT 按钮 → 点 Upload → 看到 Connecting 后松开
  • 部分板子需要上传时按住 BOOT + IO0 两个按钮

问题 2：Brownout detector was triggered

  • USB 供电不足。换一个电脑上 USB 3.0 口（蓝色口），或用带供电的 USB Hub

问题 3：串口乱码

  • 波特率不匹配。检查 Serial Monitor 的波特率和代码里 Serial.begin() 是否一致

问题 4：espressif32 platform not found

  • 网络问题导致平台下载失败。打开 VS Code 终端运行：pio platform install espressif32

问题 5：Mac 找不到串口

  • 运行 ls /dev/cu.* 查看可用端口，在 platformio.ini 加 upload_port = /dev/cu.SLAB_USBtoUART
💡 加库的方法

在 platformio.ini 的 [env:esp32dev] 下加 lib_deps = arduino-libraries/Arduino_LSM6DS3（示例），保存后 PlatformIO 自动下载。也可以在 PlatformIO Home → Libraries 搜索。
