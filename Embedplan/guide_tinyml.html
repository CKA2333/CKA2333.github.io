TinyML 是什么 — 你的核心竞争力
5 分钟

TinyML = 在极低功耗的微控制器（MCU）上运行机器学习推理。不需要联网，不需要云端服务器，模型直接跑在设备上。

传感器数据
IMU / 麦克风 / ADC
→
特征提取
DSP / FFT / MFCC
→
神经网络
训练好的模型
→
分类结果
在 MCU 上输出

典型应用（你能在 ESP32 上实现的）：

  • 手势识别（加速度计数据 → 5 种手势）
  • 关键词检测（"Hey Siri" 类似的唤醒词）
  • 异常检测（震动/声音异常报警）
  • 活动识别（走路/跑步/静止）
🎯 面试加分

能说出"我在 ESP32 上部署了手势识别模型，模型量化后 50KB，推理延迟 8ms，精度 94%"，在 Apple Sensing / Nordic / Qualcomm 面试中效果极强。

数据采集（两种方案：免硬件 vs 真实传感器）
1–2 小时采集

方案 A：用手机采集（不需要额外硬件，立即开始）

  • Edge Impulse 项目 → Data Acquisition → 点击 "Connect using your mobile phone"
  • 手机扫描 QR 码，打开链接
  • 手机可以作为加速度计数据源，直接用内置传感器采集
  • 标签：gesture_circle / gesture_up_down / idle（至少 3 类）
  • 每类采集 2 分钟数据（约 120 个 2 秒样本），质量比数量重要

方案 B：ESP32 + 加速度计（更贴近真实项目）

ESP32 本身没有 IMU，需要外接 MPU6050（$1–2，AliExpress）或 ADXL345。

platformio.ini 添加库：

[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
lib_deps =
  electroniccats/MPU6050@^1.3.0
  https://github.com/edgeimpulse/arduino-data-forwarder

安装 Edge Impulse CLI（免费）：

# 需要先装 Node.js (nodejs.org, 免费)
npm install -g edge-impulse-cli

# 连接 ESP32，运行数据转发
edge-impulse-data-forwarder
⚠️ 建议先用手机方案

初次使用 Edge Impulse，先用手机采集数据跑通完整流程，再考虑连硬件。方案 A 今天就能有结果。

4
创建 Impulse（定义处理流程）
15 分钟

采集完数据后，点击左侧 "Impulse design"，这是整个 pipeline 的配置页面：

  1. Input block：Time series data，Window size = 2000 ms，Window increase = 200 ms（80% overlap，数据增强效果）
  2. Processing block：点 "+ Add a processing block"，选择：
    • Spectral Analysis（推荐）：提取 FFT 频域特征，适合周期性手势
    • 或 Flatten：直接用原始统计值（均值/方差），更简单
  3. Learning block：选 "Classification"（Neural Network），类别数填你的类别数量
  4. 点 "Save Impulse"
🧠 DSP Block vs ML Block

DSP Block（Spectral Analysis）：确定性算法，把原始时间序列变成频域特征向量。ML Block（Neural Network）：训练一个分类器来区分这些特征向量。两阶段设计让 MCU 上特征提取可以用 CMSIS-DSP 加速。

5
配置 DSP 特征提取块
10 分钟

点击左侧 "Spectral features"：

  • Scale axes：默认 1，不用改
  • Type：FFT（频域，推荐），或 Wavelet（信号局部特征）
  • FFT length：32 或 64（越大特征越细，计算越贵）
  • 点击 "Save parameters" 然后 "Generate features"

生成后会看到 Feature Explorer，一个 2D/3D 散点图显示不同类别的特征分布。如果不同类的点分得很开，模型会训练得很好。

💡 特征分不开怎么办？

① 采集更多数据（每类 5 分钟以上）② 动作做得更夸张一点 ③ 换 Processing block（试试 Raw data 或 Flatten）

6
训练神经网络
10–20 分钟

点击左侧 "NN Classifier"：

  • Training cycles：100（默认），初次试跑够了
  • Learning rate：0.0005（默认）
  • Validation set size：20%（默认）
  • 网络架构默认是 2 层全连接（足够手势识别），点击 "Start training"

训练完成后看 Accuracy 和 Confusion Matrix：

  • Accuracy > 90%：很好，可以部署
  • Accuracy 70–90%：看混淆矩阵，哪两类容易混淆，针对性补数据
  • Accuracy < 70%：数据质量问题，重新采集更均匀的数据
📊 Edge Impulse 自动量化

训练结束后，Edge Impulse 会自动显示 int8 量化后的精度对比（float32 vs int8）。一般精度损失 < 2%，但模型大小减少 4×，这就是 TinyML 的核心魔法。

7
测试和优化
30 分钟

点击 "Model testing"，用测试集评估真实性能：

  • 点击 "Classify all"，等待结果
  • 查看每个样本的预测结果，找出哪些错了
  • 回到 Data acquisition，给错误的类别补充数据，重新训练

优化小技巧：

  • 用 EON Tuner（左侧菜单）：Edge Impulse 自动搜索最优模型架构，找到精度/大小最佳平衡点
  • 调整 Window size：手势时间长的用 3000ms，快速手势用 1000ms
  • 增加 Training cycles 到 300：通常能提升 2–5% 精度

部署到 ESP32（Arduino Library）
30 分钟

点击左侧 "Deployment"：

  • 搜索 "Arduino library"，点击 Build
  • 下载 .zip 文件（约 200–500 KB）

在 PlatformIO 项目中使用：

; platformio.ini 中添加本地库路径
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
lib_deps = symlink:///path/to/gesture-recognition-arduino  ; 或直接复制到 lib/ 目录

把 .zip 解压到项目 lib/ 目录，然后写推理代码：

#include <Arduino.h>
#include <gesture-recognition_inferencing.h>  // Edge Impulse 生成的头文件

// 推理所需的输入缓冲区（大小由 EI_CLASSIFIER_DSP_INPUT_FRAME_SIZE 定义）
float features[EI_CLASSIFIER_DSP_INPUT_FRAME_SIZE];
int feature_ix = 0;

void setup() {
  Serial.begin(115200);
  Serial.printf("Model: %s\n", EI_CLASSIFIER_PROJECT_NAME);
  Serial.printf("Classes: %d\n", EI_CLASSIFIER_LABEL_COUNT);
}

void loop() {
  // 假设你已经采集了一个 window 的加速度数据到 features[]
  // （实际项目里从 MPU6050 读取）

  // 创建推理输入信号
  signal_t signal;
  numpy::signal_from_buffer(features, EI_CLASSIFIER_DSP_INPUT_FRAME_SIZE, &signal);

  // 运行推理
  ei_impulse_result_t result;
  EI_IMPULSE_ERROR err = run_classifier(&signal, &result, false);

  if (err != EI_IMPULSE_OK) {
    Serial.printf("Classifier error: %d\n", err);
    return;
  }

  // 打印结果
  Serial.printf("Inference time: %d ms\n", result.timing.classification);
  for (int i = 0; i < EI_CLASSIFIER_LABEL_COUNT; i++) {
    Serial.printf("  %s: %.2f\n",
      result.classification[i].label,
      result.classification[i].value);
  }

  delay(500);
}
💡 推理速度

ESP32 上典型推理时间：简单手势模型 5–15ms，含 FFT 的完整 pipeline 15–50ms。远快于人体反应速度（200ms），完全实时。

TFLite Micro 基础（理解底层）
1–2 小时阅读

Edge Impulse 底层用的就是 TFLite Micro。理解底层帮助你面试时讲清楚原理：

  • TFLite Micro = TensorFlow Lite 的 MCU 版本，不依赖 OS，不动态分配内存
  • Tensor Arena：一块静态分配的 RAM，供推理时所有中间张量使用。大小 = 你需要手动调整的参数
  • OpResolver：注册模型用到的算子（Conv2D、FullyConnected 等），不用的不注册，节省 Flash

Hello World 例子（TFLite 官方，免费）：

  • 教程：tensorflow.org/lite/microcontrollers
  • 这个 Hello World 训练一个 sin(x) 拟合网络，推理输出控制 LED 亮度
  • 跑通后你就理解了：模型文件（.tflite 转 .cc 数组）→ 加载 → Tensor Arena 分配 → 推理 → 读取输出
// TFLite Micro 推理流程（概念代码）
#include "tensorflow/lite/micro/micro_interpreter.h"

const int kTensorArenaSize = 20 * 1024;  // 20KB，按需调整
uint8_t tensor_arena[kTensorArenaSize];  // 静态分配（关键！MCU 没有 new/malloc）

tflite::MicroInterpreter interpreter(model, resolver,
                                     tensor_arena, kTensorArenaSize);
interpreter.AllocateTensors();  // 在 arena 里分配各层张量

TfLiteTensor* input  = interpreter.input(0);
TfLiteTensor* output = interpreter.output(0);

input->data.f[0] = x;           // 填充输入
interpreter.Invoke();            // 运行推理
float result = output->data.f[0]; // 读取输出

int8 量化实战（面试必讲）
1 小时实操

量化是 TinyML 最核心的技术，把 float32 模型压缩 4× 并加速推理：

在 Google Colab 实现量化（免费 GPU）：

import tensorflow as tf
import numpy as np

# 假设你已经训练了一个 Keras 模型 model
# 方法 1: Post-Training Quantization（训练后量化，最简单）
converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]

# 提供校准数据集（代表性输入，100–500 个样本）
def representative_dataset():
    for sample in calibration_data[:100]:
        yield [sample.reshape(1, -1).astype(np.float32)]

converter.representative_dataset = representative_dataset
converter.target_spec.supported_ops = [tf.lite.OpsSet.TFLITE_BUILTINS_INT8]
converter.inference_input_type  = tf.int8
converter.inference_output_type = tf.int8

tflite_model = converter.convert()

# 保存
with open('model_int8.tflite', 'wb') as f:
    f.write(tflite_model)

print(f"Float32 size: {len(tf.lite.TFLiteConverter.from_keras_model(model).convert())/1024:.1f} KB")
print(f"Int8 size:    {len(tflite_model)/1024:.1f} KB")
# 典型输出：Float32 size: 200.0 KB → Int8 size: 52.0 KB（4× 压缩）

把 .tflite 转成 C 数组（烧进 Flash）：

# 在终端运行（需要 xxd 工具，Linux/Mac 自带，Windows 用 WSL）
xxd -i model_int8.tflite > model_data.cc

# 生成的 model_data.cc 包含：
# const unsigned char g_model[] = { 0x1c, 0x00, 0x00, 0x00, ... };
# const int g_model_len = 12345;
💡 量化精度损失实测

手势识别：float32 精度 96.2% → int8 精度 95.8%（损失 0.4%，完全可接受）。模型大小：float32 180KB → int8 48KB。ESP32 Flash 4MB，完全放得下。
